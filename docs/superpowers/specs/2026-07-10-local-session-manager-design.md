# 本机会话管理设计

## 目标

解决切换 `CODEX_HOME` 后旧会话不可见的问题。CodexMonitor 聚合当前 Windows 用户使用过的多个 Codex 数据目录，提供统一浏览、固定范围内容搜索、继续原会话、派生到当前任务、归档和安全永久清理能力。

## 范围

### 包含

- 顶部侧栏会话管理入口和独立管理模式。
- 当前用户多个 `CODEX_HOME` 来源管理。
- 活跃及归档主会话聚合展示。
- 子 Agent 默认隐藏，可通过筛选显示。
- 标题、ID、项目名、路径、用户消息、Agent 最终回复搜索。
- 继续原会话、派生到当前任务、归档、复制 ID、打开目录。
- 多选归档。
- 已归档会话手动永久删除。
- 按归档时间自动永久删除。
- 删除审计记录。

### 不包含

- 扫描其他 Windows 用户账户。
- 推理、工具调用、命令输出搜索。
- 高级搜索范围配置。
- 物理修改原会话 `cwd`。
- 批量派生或批量迁移。
- 自动级联删除子 Agent。
- 模糊匹配或递归删除会话目录。

## 会话来源

`SessionSource` 表示一个当前用户可访问的 Codex 数据目录。

来源发现：

1. 当前设置中的 `CODEX_HOME`。
2. 默认 `~/.codex`。
3. CodexMonitor 历史成功使用过的 `CODEX_HOME`。
4. 用户手动添加的目录。

来源规则：

- 自动记录成功使用过的目录。
- 切换当前配置不会移除历史来源。
- 支持添加、重命名、停用、移除、重新扫描。
- 移除来源只移除管理记录，不删除磁盘数据。
- 当前来源和默认来源保留系统标识。
- 无访问权限或目录不存在时标记不可用，不阻塞其他来源。
- 不扫描 `C:\Users` 下其他 Windows 账户。
- 用户手动添加其他可访问目录时，将其视为显式授权来源。

建议数据：

```ts
type SessionSource = {
  id: string;
  name: string;
  codexHomePath: string;
  enabled: boolean;
  isCurrent: boolean;
  isDefault: boolean;
  discoveredAt: number;
  lastScanAt: number | null;
  status: "ready" | "missing" | "denied" | "invalid" | "scanning";
  error: string | null;
};
```

来源路径必须规范化后去重。Windows 路径比较使用不区分大小写的规范化键。会话全局唯一键为 `sourceId + threadId`。

## 管理模式

侧栏顶部在主页与排序按钮之间增加会话管理图标。

行为：

- 点击进入 `sessionManager` 模式，图标高亮。
- 再次点击、点击主页或进入普通项目会话时退出。
- 复用现有侧栏视觉、搜索框、会话行、时间展示与设计系统组件。
- 不复用普通侧栏的数据、分页、搜索、展开和滚动状态。
- 普通侧栏状态退出后完整恢复。
- 管理模式拥有独立搜索、筛选、分页、多选和滚动状态。
- 管理模式刷新按钮只刷新会话索引。
- 打开或继续会话成功后退出管理模式。

## 会话索引

采用混合架构。

- Rust 直接安全扫描各来源会话文件，负责索引、搜索和永久删除定位。
- Codex app-server 负责继续、恢复和归档等官方协议操作。
- 每个来源的 app-server 按需启动，空闲后关闭，不常驻全部来源进程。

建议索引记录：

```ts
type ManagedSession = {
  key: string;
  sourceId: string;
  threadId: string;
  cwd: string | null;
  title: string;
  preview: string | null;
  createdAt: number | null;
  updatedAt: number | null;
  archivedAt: number | null;
  isArchived: boolean;
  parentThreadId: string | null;
  isSubagent: boolean;
  subagentNickname: string | null;
  subagentRole: string | null;
  projectExists: boolean;
  fileStatus: "mapped" | "unmapped" | "missing" | "invalid";
};
```

扫描规则：

- 每个启用来源独立扫描，单一来源失败不影响其他来源。
- 同时扫描来源数限制为 2–4。
- 解析异常文件跳过并记录诊断。
- 不因索引扫描启动或恢复会话。
- 默认隐藏子 Agent；筛选开启后显示并标记父会话、昵称、角色。
- 活跃和归档会话均进入索引。
- 目录已删除的项目会话仍显示并标记项目不存在。

## 搜索

固定搜索范围：

- 标题。
- 会话 ID。
- 项目名。
- 项目路径。
- 用户消息。
- Agent 最终回复。

永久排除：

- 推理内容。
- 工具调用。
- 命令输出。
- 审批内部状态。

执行：

1. 摘要字段在内存索引中即时匹配。
2. 输入至少两个字符后启动内容扫描。
3. 按来源、文件分批读取，逐批返回结果与进度。
4. 关键词改变时取消旧任务。
5. 缓存键为来源、会话 ID、文件修改时间。
6. 缓存仅保留规范化搜索文本、命中片段和元数据，不保留排除内容。
7. 首版缓存为进程内缓存，不增加数据库依赖。
8. 超大文件或不完整解析必须标记“未完全搜索”，不得静默宣称无结果。
9. 搜索不会连接项目、恢复会话或改变归档状态。

## 会话操作

### 继续原会话

- 使用会话所属来源的 `CODEX_HOME` 按需启动 app-server。
- 沿用原 `threadId` 和完整可恢复上下文。
- 根据会话 `cwd` 定位或创建 CodexMonitor 工作区。
- 后续请求绑定原来源，不修改全局 `CODEX_HOME`。
- UI 显示来源标签。
- 归档会话由 app-server 的恢复/取消归档行为处理；失败时保留管理列表状态并提示。

### 派生到当前任务

- 在当前工作区创建新会话和新 `threadId`。
- 提取原会话标题、用户消息和 Agent 最终回复。
- 生成结构化交接内容作为新会话首条上下文。
- 记录来源会话键，供 UI 展示派生关系。
- 不复制推理、工具输出、历史审批状态。
- 原会话和文件保持不变。

### 归档

- 通过会话来源对应 app-server 调用官方归档协议。
- 归档成功后更新索引，并记录 `archivedAt`。
- 多选只支持批量归档；单条失败不阻塞其他条目。
- 普通“删除”文案继续代表归档，不代表永久删除。

## 归档时间账本

永久删除必须按归档时间，不按会话最后更新时间。

归档时间来源优先级：

1. Codex 会话元数据提供的可靠归档时间。
2. CodexMonitor 成功执行归档时记录的时间。
3. 首次发现旧归档会话的时间。

旧会话首次发现规则：

- 无可靠归档时间时，将首次发现时间作为清理计时起点。
- 首次发现时绝不立即永久删除。
- 账本按 `sourceId + threadId` 存储。

## 手动永久删除

仅对已归档会话开放。

安全流程：

1. 用户选择永久删除。
2. 展示不可恢复警告、来源、会话 ID、归档时间。
3. 父会话显示关联子 Agent 数量。
4. 默认仅删除父会话。
5. 可选“同时永久删除关联子 Agent”，默认不勾选。
6. 二次确认后执行。
7. 删除前再次验证来源根目录、文件映射、thread ID、归档状态和归档时间。
8. 只允许删除来源会话/归档目录内已验证的精确文件。
9. 禁止目录递归删除、字符串模糊匹配和越界符号链接。
10. 找不到可靠映射时跳过并提示，不猜测文件位置。
11. 删除成功后刷新管理索引和普通会话列表。

自动删除不级联。父会话和子 Agent 按各自归档时间独立判断。

## 自动永久删除

设置：

```ts
autoDeleteArchivedThreadsEnabled: boolean; // 默认 false
autoDeleteArchivedThreadsDays: 30 | 60 | 90 | 180; // 默认 30
```

开启流程：

1. 开关默认关闭。
2. 用户手动点击开启。
3. 弹出明确说明：永久删除且无法恢复。
4. 扫描并展示预计符合条件数量。
5. 用户进行第二次明确确认。
6. 确认后才保存开启状态。
7. 取消时保持关闭，不执行扫描后的删除。
8. 首次开启只预览，不立即删除。

执行时机：

- 应用启动且会话来源可用后检查一次。
- 后台运行期间每 24 小时最多检查一次。
- 首次实际自动清理发生在下次启动或开启满 24 小时后。
- 设置页提供“立即清理”，每次都需再次二次确认。

删除前排除：

- 未归档会话。
- 归档时间未超期。
- 当前打开会话。
- 运行中会话。
- 置顶会话。
- 无可靠文件映射会话。
- 来源不可用会话。

处理：

- 每条会话独立处理。
- 单条失败不阻塞其余会话。
- 应用退出中断时，下次重新校验并继续。
- 关闭开关后停止新的自动清理调度。

## 审计日志

只记录：

- 来源 ID 和规范化来源路径标识。
- 会话 ID。
- 归档时间。
- 删除时间。
- 手动或自动原因。
- 删除结果和错误摘要。
- 是否请求级联子 Agent。

不记录会话正文、推理或工具输出。

## 后端运行时

新增来源绑定的管理运行时：

- `sourceId` 解析到规范化 `CODEX_HOME`。
- app-server 会话键不能只用 `workspaceId`，需包含来源身份。
- 恢复后的线程请求持续路由到原来源。
- 来源 app-server 按需创建，记录最后活动时间，空闲后安全关闭。
- 同一来源可复用运行时；不同来源不得串用认证、配置或会话目录。
- app/daemon 两个运行面需保持 RPC 契约一致。

## 界面状态

建议：

```ts
type SidebarMode = "workspaces" | "sessionManager";
```

管理状态独立存储：

- `query`
- `showSubagents`
- `statusFilter`
- `sourceFilter`
- `selectedSessionKeys`
- `cursorBySource`
- `searchProgress`
- `scrollOffset`

普通侧栏状态不得被管理模式重置。

## 错误与降级

- 来源不可用：显示来源错误，其他来源继续。
- app-server 启动失败：列表仍可浏览和搜索；继续/归档禁用或失败提示。
- 文件格式变化：无法解析条目标记诊断，不允许永久删除。
- 内容搜索取消：保留已返回结果并标记已停止。
- 删除部分失败：展示成功/失败数量和可重试条目。

## 分期

### 第一期：来源、索引、浏览和搜索

- 来源持久化与设置 UI。
- 安全文件扫描和会话索引。
- 管理模式 UI。
- 摘要搜索和按需内容搜索。
- 子 Agent 显示筛选。

### 第二期：来源运行时与会话操作

- 来源 app-server 生命周期。
- 继续原会话。
- 归档和批量归档。
- 派生到当前任务。
- 来源标签和错误状态。

### 第三期：永久清理

- 归档时间账本。
- 精确文件映射和手动永久删除。
- 自动永久删除设置、预览、调度。
- 审计日志和部分失败报告。

## 验收标准

- 切换当前 `CODEX_HOME` 后，历史来源会话仍在管理模式可见。
- 来源间相同 thread ID 不冲突。
- 内容搜索不匹配推理、工具调用和命令输出。
- 继续旧来源会话不修改全局 `CODEX_HOME`。
- 派生操作创建新会话且不修改原会话。
- 自动永久删除默认关闭，未经二次确认不能开启。
- 自动删除按归档时间且首次发现旧归档不会立即删除。
- 永久删除不能越出来源会话目录，不能删除映射不可靠的文件。
- 单来源或单会话失败不阻塞其他来源/会话。
